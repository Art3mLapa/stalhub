# Contributing to the StalHub Project

## 📌 General Guidelines

We welcome contributions of all kinds, including:

- Bug fixes  
- Typo corrections  
- Code formatting improvements  
- Refactoring that improves code structure without changing functionality  
- New features (see below)

---

## 🧩 Creating New Features

Adding new functionality can be a complex and time-consuming task.

You are welcome to contribute small features that do not introduce breaking changes to the API or core backend logic.

If you have an idea for a larger feature, **please open a [GitHub issue](https://github.com/oarer/stalhub/issues/new)** so we can discuss it before implementation ❤️

---

## 🎨 Visual Changes

When making visual (CSS/design) changes, please ensure they align with the overall style of the project.

Avoid changes that break visual consistency.

If you're unsure — it's better to start a discussion in an **[issue](https://github.com/oarer/stalhub/issues/new)** before proceeding.

---

## 🌐 Network-Related Changes

Please avoid introducing dependencies on third-party or personal APIs.

Pull requests introducing such dependencies may be declined to keep the project stable and self-contained.

---

## 🧾 Code Style

Please follow the existing code style used in the project.

Use the provided `.prettier` or `biome.json` configuration files to format your code before submitting a pull request.

---

## 🚫 What Is Not Welcome

- Removing existing features without a clear reason  
- Adding features or content that violate the rules of platforms relevant to the project (e.g., Twitch, Discord)  
- Irrelevant content unrelated to the theme of the project  
- Phishing links, ads, IP loggers, backdoors, or any harmful code  
- Functionality that bypasses restrictions or accesses private user data  
- Meaningless commits (e.g. random copy-paste or spam)  
- Editing global configuration, service, or deployment workflow files without prior discussion  

---

## 💬 Need Help?

If you're unsure about anything — feel free to open an issue or start a discussion.

It's always better to clarify before submitting a PR.

---

## 🔗 Related Repositories

This repository contains the frontend of the project.

- Backend: https://github.com/oarer/stalhub_backend

Some features may require changes in both repositories.  
If you're unsure — feel free to open an issue first.
